# 退神股票涨跌理论与当前量化实现差距分析

本文对照 `docs/theory/退神股票涨跌理论.md` 和当前 `src/market_regime_alpha/dividend_t` 代码，分析退神股票涨跌理论中的逻辑在当前量化程序中的实现程度。

结论先行：

```text
当前程序实现了“退神变量的红利做 T 简化版”。

它已经有：
F/R/T 评分、BUY_T / SELL_T / STOP_T / REDUCE 等信号、基础风控和 Web Dashboard。

它还没有完整实现：
买量 / 卖量的真实比值模型、市场关注度八点属性、涨跌确定性的六大判断依据、特定情境记忆、动态权重、卖出率、最大可卖量的市场结构估算。
```

## 1. 总纲公式覆盖情况

退神理论核心公式：

```text
买量 = 市场关注资金量 × 购买率
市场关注资金量 = 市场关注度 × 人均关注资金量
购买率 ≈ 涨的确定性 × 盈亏比

卖量 = 最大可卖量 × 卖出率
卖出率 ≈ 不涨的确定性 × 亏盈比
```

当前实现：

```python
R = 0.25G + 0.30Z + 0.25K + 0.20(5 - S)
```

覆盖程度：部分实现。

已经实现：

- `G_score`：市场关注度的人工评分。
- `Z_score`：上涨确定性的人工评分。
- `K_score`：由盈亏比映射成 0-5 分。
- `S_score`：卖压的人工评分。
- `R_score`：把 G/Z/K/S 合成为 0-100 的退神评分。

主要差距：

- 当前 `R_score` 是加权评分，不是买量 / 卖量比值。
- 没有拆出 `市场关注度 × 人均关注资金量`。
- 没有真实计算 `购买率`。
- 没有真实计算 `卖出率`。
- 没有不涨确定性、亏盈比。
- `S_score` 只是人工卖压分数，不是市场最大可卖量。

## 2. 市场关注度八点属性覆盖情况

退神理论把市场关注度拆成八点：

```text
逐利导向
映入机会
增减原理
分流现象
正负反馈
快耗现象
资金差异
宏观影响
```

当前实现：只实现为一个人工输入字段。

对应代码：

```text
RetreatInputs.market_attention
ScoreBreakdown.G_score
```

缺失内容：

- 没有用成交额、换手率、涨跌幅榜、新闻公告、板块排名来计算关注度。
- 没有记录关注度的上升、下降、加速度。
- 没有同类标的分流逻辑，例如红利股、煤炭、电力、银行之间的资金切换。
- 没有正反馈模型，例如放量上涨后吸引更多买盘。
- 没有快耗模型，例如突然放量后关注资金短期耗尽。
- 没有区分投资者类型和人均资金量。
- 没有宏观市场热度、指数成交额、风格切换、流动性环境。

## 3. 涨与不涨确定性覆盖情况

退神理论中的六大判断依据：

```text
业绩向好性
同质同价
形态的大概率
市场关注度
最大可卖量
特定情境记忆
```

当前实现：部分覆盖。

已经覆盖：

- 业绩向好性：用 `FundamentalInputs` 的分红、估值、周期、财务、催化粗略表示。
- 同质同价：用 `valuation_margin` 粗略表示。
- 形态的大概率：用 `TechnicalInputs` 和 `indicators.py` 的均线、支撑、压力、成交量粗略表示。
- 市场关注度：用 `market_attention` 粗略表示。
- 最大可卖量：用 `sell_pressure` 粗略表示。

没有覆盖：

- 行业发展空间与公司脱颖而出没有拆开。
- 同质同价没有建立可比公司估值池。
- 形态的大概率没有统计历史形态成功率。
- 特定情境记忆没有实现。
- 因子发散、股性记忆、高手印象没有实现。
- 判断依据之间的联动关系没有实现。
- 判断依据的动态权重没有实现。
- 不同投资者群体的判断依据偏好没有实现。

## 4. 特定情境记忆覆盖情况

退神理论强调：

```text
P1 权重 50%
P2 权重 30%
P3 权重 15%
P4 权重 3%
并且随时间衰弱
```

当前实现：未实现。

缺失内容：

- 没有记录某个因子最近几次出现后的收益表现。
- 没有记录某只股票的股性记忆。
- 没有记录某个行业或概念的赚钱效应记忆。
- 没有时间衰减。
- 没有 P1/P2/P3/P4 权重。
- 没有把特定情境记忆反向作用到 `Z_score` 或 `G_score`。

这是当前程序相对退神理论最大的缺口之一。

## 5. 盈亏比覆盖情况

当前实现：部分覆盖。

已经实现：

- `RetreatInputs.risk_reward_ratio` 表示盈亏比。
- `risk_reward_score()` 把盈亏比映射成 K 分数。
- `indicators.estimate_levels()` 可以用支撑位、压力位估算一个技术盈亏比。

缺失内容：

- 没有按不同买入依据计算不同盈亏比。
- 没有把时间成本纳入亏的一侧。
- 没有把分红收益、除权、股息税、持有期纳入红利股盈亏比。
- 没有区分趋势形态强度、趋势形态偏离、短期形态强度、短期形态偏离。
- 没有把投资者风险偏好纳入购买率。
- 没有实现确定性与盈亏比之间的非线性关系。

## 6. 最大可卖量覆盖情况

当前实现：弱覆盖。

已有字段：

```text
RetreatInputs.sell_pressure
PositionState.available_sell_pct
```

但要注意：

```text
sell_pressure 是市场卖压评分。
available_sell_pct 是账户里自己的可卖仓位。

两者都不是退神理论中的“市场最大可卖量”。
```

缺失内容：

- 没有计算流通市值、自由流通盘。
- 没有计算成交密集区。
- 没有计算前高套牢盘。
- 没有计算短期获利盘。
- 没有计算融资盘压力。
- 没有计算解禁、减持、除息兑现盘。
- 没有用量价分布估算筹码结构。
- 没有卖出率。

## 7. 买入依据组合与动态权重覆盖情况

退神理论列出了多种买入依据组合，例如：

```text
业绩向好性 + 形态的大概率
同质同价 + 形态趋势超跌
因子发散 + 形态短期走强
市场关注度 + 形态短期走强
```

当前实现：只实现了一个组合。

当前组合可以理解为：

```text
红利基本面
+
退神 G/Z/K/S 人工评分
+
技术位置
+
仓位风控
```

缺失内容：

- 没有策略模板系统。
- 没有为不同买入依据组合定义不同权重。
- 没有根据市场阶段自动切换权重。
- 没有短线题材、因子发散、股性记忆、高手印象等模式。

这不是当前程序的 bug，而是因为当前目标本来是“长期 / 红利 / 做 T”，不是完整短线涨跌理论平台。

## 8. 风控与执行覆盖情况

当前实现较完整的部分：

- F < 50 清仓。
- F < 55 减底仓。
- F < 60 停止做 T。
- 下跌趋势停止做 T。
- 板块走弱停止做 T。
- 连续两次 T 失败停止做 T。
- 单票仓位上限。
- 周期股仓位上限。
- 单次主动买入上限，并区分底仓目标、主动增量上限和最大总仓位。
- 最低现金保留。
- A 股 T+1 可卖仓位约束。

主要缺失：

- 没有真实交易账户同步。
- 没有 QMT / PTrade 实盘委托、撤单、成交回报。
- 没有订单状态机。
- 没有重复下单防护。
- 没有涨跌停、停牌、集合竞价约束。
- 没有滑点、手续费、股息税。
- 没有模拟盘撮合。

## 9. 数据层覆盖情况

当前已有：

- Tushare 日线、分钟线查询基础。
- Tushare 日线、每日指标、复权因子、分红、财务指标入口预留。
- DuckDB + Parquet 存储接口预留。
- 红利股观察池 CSV。

缺失：

- 没有每日自动同步 Tushare 数据。
- 没有把财务、分红、复权、行情合并成特征表。
- 没有板块指数、红利指数、行业指数。
- 没有 ETF 资金流。
- 没有新闻公告和事件数据。
- 没有 QMT / PTrade 实时行情接入。
- 没有账户持仓与本地状态同步。

## 10. 推荐补齐顺序

### 第一步：把人工评分改成可解释特征

优先实现：

```text
G：成交额分位、换手率分位、行业相对强度、红利指数强弱。
Z：分红可持续性、估值分位、利润趋势、行业景气、运价/煤价/电价等行业锚。
K：支撑压力、ATR 止损、上方空间、下方失效位。
S：前高压力、成交密集区、短期获利盘、流通市值、除息/解禁/减持事件。
```

### 第二步：把 R_score 从加权分改成买卖力量比

目标形式：

```text
buy_pressure = G_amount * purchase_rate
purchase_rate = f(Z, K, risk_preference)

sell_pressure_amount = S_amount * sell_rate
sell_rate = f(not_up_certainty, loss_profit_ratio)

R_value = buy_pressure / max(sell_pressure_amount, epsilon)
```

### 第三步：实现特定情境记忆

需要新增状态表：

```text
factor_memory
stock_memory
sector_memory
pattern_memory
```

最小版本：

```text
最近 4 次同类信号后的 1/3/5/10 日收益
P1/P2/P3/P4 加权
随时间衰减
输出 memory_score
```

### 第四步：接入回测与复盘

必须先回测：

```text
正 T 买入是否真的降低成本？
倒 T 是否经常踏空？
F < 60 停手是否能减少下跌趋势损失？
K >= 2 是否比 K < 2 有明显优势？
S 上升是否对应后续收益下降？
```

### 第五步：再接 QMT / PTrade

顺序必须是：

```text
只读行情和账户
模拟订单
人工确认下单
小资金实盘
自动化约束订单
```

## 11. 当前结论

当前程序可以作为学习和研究平台继续扩展，但不能说已经完整实现退神股票涨跌理论。

更准确的描述是：

```text
当前实现 = 红利做 T 模型的可运行骨架
理论实现程度 = 核心变量命名和部分阈值规则已实现
主要缺口 = 变量自动计算、动态反馈、特定情境记忆、真实买卖力量比、卖量结构、回测验证
```

下一步最值得做的不是接实盘交易，而是：

```text
先把 G/Z/K/S 从人工输入改成可复现的数据特征，
再用历史数据验证这些特征是否真的能解释后续涨跌和 T 仓胜率。
```
