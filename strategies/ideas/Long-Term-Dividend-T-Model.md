# 长期 / 红利 / 做 T 模型设计文档

> 本文是学习和策略研究文档，不构成投资建议或收益承诺。A 股交易规则、标的基本面、分红政策和市场环境会变化，实际使用前需要用最新数据重新验证。

## 0. 模型定位

本文档设计的是一套适用于 **中远海控、煤炭、电力、银行、运营商、港口、高速、部分央国企红利股** 的交易模型。

核心结构是：

```text
长期 / 红利 / 做 T
=
基本面逻辑
+
退神理论
+
技术位置
```

它不是短线追涨模型，也不是纯价值投资模型，而是：

> **用基本面决定是否值得长期持有，用退神理论判断资金买卖力量，用技术位置决定 T 仓买卖点。**

## 1. 模型总目标

### 1.1 交易目标

这套模型的目标不是每天抓涨停，而是：

```text
1. 选择长期逻辑相对稳定的红利股。
2. 建立底仓，吃股息、估值修复、周期波动。
3. 利用 T 仓在区间波动中高抛低吸。
4. 降低持仓成本。
5. 避免因短线波动丢掉长期底仓。
6. 避免在趋势破坏后机械补仓。
```

### 1.2 模型适用场景

适合：

```text
红利股
高股息资产
周期性龙头
央国企估值修复
行业龙头区间震荡
有分红锚、估值锚、行业锚的股票
```

不适合：

```text
纯题材小票
连续涨停接力票
无业绩支撑的概念股
高波动庄股
财务不透明股票
流动性差的小盘股
```

### 1.3 底层假设

这套模型基于三个假设：

```text
1. 红利股的中长期价格有基本面锚。
2. 红利股短期仍然受资金买卖力量影响。
3. 技术位置可以帮助优化买卖时机，但不能替代基本面判断。
```

官方投资者教育材料也强调，股票等有市场风险的资产无法保证收益，过度依赖收益预测会形成不现实预期；分散投资也不能消除亏损，但有助于降低单一资产风险。这个模型因此必须内置仓位和风控，而不是只设计买卖信号。([Investor.gov][1])

## 2. 模型总架构

整体架构如下：

```text
第一层：基本面逻辑层
    决定：这只股票值不值得做底仓？

第二层：退神理论层
    决定：当前买量、卖量、确定性、盈亏比如何变化？

第三层：技术位置层
    决定：今天是否适合买 T？是否适合卖 T？

第四层：仓位执行层
    决定：底仓多少？T 仓多少？现金多少？

第五层：风控退出层
    决定：什么情况下停止做 T？什么情况下减底仓？
```

可以用一句话表示：

```text
基本面定资格，
退神理论定资金行为，
技术位置定买卖点，
仓位系统定风险。
```

## 3. 模型核心原理

### 3.1 基本面逻辑：决定“能不能长期拿”

红利股不能只看股价波动，首先要判断它是否值得作为底仓。

基本面逻辑回答的是：

```text
这家公司未来还能不能赚钱？
现金流是否稳定？
分红是否可持续？
当前估值是否合理？
行业周期是否处于可接受区间？
```

对于中远海控这类周期红利股，基本面逻辑尤其重要。

因为它不是银行、电力这种利润相对稳定的红利资产，而是带有强周期属性：

```text
运价上升 -> 利润改善 -> 分红预期增强 -> 估值修复
运价下降 -> 利润下降 -> 分红预期下降 -> 估值承压
```

所以它的底仓逻辑不能只看股息率，还要看：

```text
航运景气度
运价指数
公司成本
现金储备
分红政策
全球贸易需求
地缘冲突对航线的影响
```

### 3.2 退神理论：判断“现在谁在买，谁在卖”

退神理论的核心可以压缩成：

```text
上涨动力 ~= 买量 / 卖量
```

进一步拆成：

```text
买量 = 市场关注资金量 * 购买率

购买率 ~= 涨的确定性 * 盈亏比

卖量 = 最大可卖量 * 卖出率
```

在本模型中，退神理论对应四个变量：

| 变量 | 含义 | 红利股做 T 中的解释 |
| --- | --- | --- |
| **G：市场关注资金量** | 有多少资金正在关注 | 红利资金、避险资金、行业资金是否回流 |
| **Z：涨的确定性** | 市场相信它会上涨的程度 | 分红预期、业绩预期、运价预期是否增强 |
| **K：盈亏比** | 当前位置划不划算 | 距离支撑近不近，距离压力远不远 |
| **S：最大可卖量** | 潜在卖压 | 前高套牢盘、获利盘、解禁、除息兑现盘 |

做 T 的核心就是判断：

```text
低位时：
S 是否下降？
K 是否变好？
Z 是否还在？
G 是否有回流迹象？

高位时：
S 是否上升？
K 是否变差？
G 是否耗尽？
Z 是否已经被充分交易？
```

### 3.3 技术位置：决定“在哪里动手”

技术位置不是用来预测长期价值，而是用来解决：

```text
今天买是不是太早？
今天卖是不是太急？
这个位置的盈亏比是否合适？
```

常用技术依据：

```text
5 日线
10 日线
20 日线
60 日线
前高
前低
箱体上沿
箱体下沿
成交密集区
放量突破
缩量回踩
高位放量滞涨
低位缩量止跌
```

技术位置只解决 **交易时机**，不解决 **股票是否值得持有**。

## 4. 仓位结构设计

A 股股票通常是 **股票 T+1、资金 T+0**：当天买入的股票通常不能当天卖出，但当天卖出股票所得资金可以继续用于买入。因此，A 股做 T 需要有底仓，所谓“做 T”本质上是用已有可卖股票和新增资金做仓位替换。([新浪财经][2])

### 4.1 仓位分层

建议把仓位拆成三层：

```text
底仓：长期持有
T 仓：短期波动交易
现金仓：防守和补位
```

示例结构：

| 仓位 | 比例 | 作用 |
| --- | ---: | --- |
| 底仓 | 50% - 70% | 吃红利、估值修复、长期逻辑 |
| T 仓 | 20% - 30% | 高抛低吸，降低成本 |
| 现金仓 | 10% - 20% | 防止被动满仓，保留机会 |

如果标的波动较大，例如周期股：

```text
底仓可以略低，T 仓和现金仓略高。
```

如果标的波动较小，例如银行、电力、公用事业：

```text
底仓可以略高，T 仓略低。
```

## 5. 做 T 类型设计

### 5.1 正 T

正 T 是：

```text
先买入，再卖出已有底仓。
```

适用场景：

```text
早盘或盘中回踩支撑位
缩量下跌
卖压衰减
分时出现承接
预计当天或次日有修复
```

操作逻辑：

```text
低位买入 T 仓
↓
股价反弹
↓
卖出等量原有底仓
↓
总持仓不变，持仓成本下降
```

正 T 的关键：

```text
必须有原有底仓可卖。
```

因为当天新买入的 A 股通常不能当天卖出。

### 5.2 倒 T

倒 T 是：

```text
先卖出部分底仓，再低位买回。
```

适用场景：

```text
高开过多
冲到压力位
放量滞涨
红利板块转弱
市场资金切向题材方向
```

操作逻辑：

```text
高位卖出部分底仓
↓
等待回落
↓
低位买回
↓
总持仓恢复，成本下降
```

倒 T 的风险：

```text
卖出后股价继续上涨，导致踏空。
```

所以倒 T 适合在明确压力位和高位放量滞涨时使用，不适合在强趋势主升阶段频繁使用。

## 6. 基本面逻辑层设计

### 6.1 基本面评分 F

基本面评分用于决定：

```text
是否允许建底仓？
底仓比例可以多大？
是否允许继续做 T？
```

评分范围：0 - 100。

| 模块 | 权重 | 说明 |
| --- | ---: | --- |
| 分红可持续性 | 25% | 分红是否来自真实利润和现金流 |
| 估值安全边际 | 20% | PE、PB、股息率、历史估值分位 |
| 行业景气度 | 20% | 行业周期是否改善 |
| 财务质量 | 20% | 负债、现金流、资产负债表 |
| 催化稳定性 | 15% | 分红预案、业绩预告、行业事件 |

公式：

```text
F = 0.25D + 0.20V + 0.20C + 0.20Q + 0.15E
```

其中：

```text
D = 分红可持续性
V = 估值安全边际
C = 行业景气度
Q = 财务质量
E = 催化稳定性
```

### 6.2 基本面等级

| F 分数 | 等级 | 操作 |
| ---: | --- | --- |
| >= 80 | 优质底仓 | 可配置较高底仓 |
| 70 - 79 | 合格底仓 | 可正常持有 |
| 60 - 69 | 观察底仓 | 降低底仓比例 |
| 50 - 59 | 弱底仓 | 只适合短线波动，不适合重仓 |
| < 50 | 不合格 | 不做底仓，不做 T |

关键原则：

```text
F < 60，不允许重仓做 T。
F < 50，停止做 T。
```

因为做 T 的前提是：

```text
这只股票值得反复操作。
```

如果基本面坏了，继续做 T 就可能变成下跌趋势中的补仓。

## 7. 退神理论层设计

### 7.1 G：市场关注资金量

G 判断的是：

```text
现在有没有资金愿意看它、买它、重新定价它？
```

红利股的 G 可以用以下指标观察：

| 指标 | 说明 |
| --- | --- |
| 成交额变化 | 是否明显放量 |
| 换手率 | 是否有资金活跃 |
| 红利指数表现 | 红利资产是否整体走强 |
| 行业板块排名 | 航运、煤炭、银行等是否强于大盘 |
| ETF 资金流 | 高股息 ETF、红利 ETF 是否流入 |
| 市场风格 | 资金偏防守还是偏成长 |
| 消息关注度 | 分红、运价、业绩、政策是否被市场讨论 |

G 的判断：

```text
G 上升 = 市场开始重新关注
G 平稳 = 区间震荡
G 下降 = 资金撤离或关注度衰退
```

### 7.2 Z：上涨确定性

Z 判断的是：

```text
市场为什么相信它会涨？
这个理由是否增强？
```

红利股的 Z 来源包括：

```text
分红预期增强
业绩预期改善
行业价格改善
估值修复逻辑
避险资金回流
中特估或红利风格强化
```

对于中远海控这类公司，Z 的核心不是单纯股息率，而是：

```text
股息率 + 运价周期 + 利润预期 + 资产负债表 + 分红政策
```

如果股价上涨，但运价、业绩、分红预期都没有改善，那么 Z 不强，只是短线资金脉冲。

### 7.3 K：盈亏比

K 判断的是：

```text
当前位置买入，赚亏是否划算？
```

可以用技术位置辅助判断。

例如：

```text
距离支撑位近，距离压力位远 -> K 好
距离支撑位远，距离压力位近 -> K 差
连续上涨后追高 -> K 差
缩量回踩关键支撑 -> K 好
```

简单公式：

```text
盈亏比 K = 预期上涨空间 / 预期下跌空间
```

例如：

```text
买入价：14.00
压力位：14.80
止损位：13.60

预期上涨空间 = 14.80 - 14.00 = 0.80
预期下跌空间 = 14.00 - 13.60 = 0.40

K = 0.80 / 0.40 = 2.0
```

一般要求：

```text
K >= 2，适合考虑买 T。
K < 1.5，谨慎。
K < 1，不买。
```

### 7.4 S：最大可卖量

S 判断的是：

```text
上面有多少人想卖？
```

红利股里的 S 主要来自：

```text
前期套牢盘
短期获利盘
除权除息前后兑现盘
机构调仓
大股东减持
周期预期转弱
市场风格切换
融资盘压力
```

观察指标：

| 信号 | 含义 |
| --- | --- |
| 高位放量滞涨 | 获利盘增强 |
| 前高附近反复冲不过 | 套牢盘重 |
| 利好公布不涨 | 买盘不足，卖盘增强 |
| 缩量阴跌 | 关注度下降 |
| 放量长阴 | 主动卖盘强 |
| 除息前后异常放量 | 分红交易兑现 |

## 8. 技术位置层设计

### 8.1 支撑区

支撑区用于正 T 买入。

常见支撑：

```text
20 日均线
60 日均线
前低
箱体下沿
缺口附近
前期放量平台
整数关口
布林带下轨
```

有效支撑需要满足：

```text
1. 股价接近支撑位。
2. 下跌缩量。
3. 分时有承接。
4. 板块没有系统性破位。
5. 基本面逻辑没有变化。
```

### 8.2 压力区

压力区用于卖 T 或倒 T。

常见压力：

```text
前高
箱体上沿
60 日线下压位置
前期成交密集区
跳空缺口
短期涨幅过大区域
布林带上轨
```

有效压力需要满足：

```text
1. 股价接近压力位。
2. 放量但涨不动。
3. 分时冲高回落。
4. 板块开始分化。
5. 盈亏比明显变差。
```

### 8.3 趋势状态

技术层需要先判断当前处于哪种趋势：

| 趋势状态 | 特征 | 操作 |
| --- | --- | --- |
| 上升趋势 | 均线多头，回踩不破 | 做正 T 为主，少倒 T |
| 区间震荡 | 箱体上下沿清晰 | 高抛低吸 |
| 下跌趋势 | 均线空头，反弹无力 | 停止做 T 或只减仓 |
| 突破阶段 | 放量突破平台 | 不急卖 T，观察持续性 |
| 高位衰竭 | 放量滞涨，冲高回落 | 减 T 仓或倒 T |

## 9. 交易决策规则

### 9.1 底仓规则

底仓由基本面评分决定。

| 基本面评分 F | 底仓上限 |
| ---: | ---: |
| >= 80 | 70% |
| 70 - 79 | 60% |
| 60 - 69 | 40% |
| 50 - 59 | 20% |
| < 50 | 0% |

解释：

```text
基本面越强，允许底仓越高。
基本面越弱，越不能用做 T 掩盖风险。
```

### 9.2 正 T 买入规则

满足以下条件，可以考虑正 T：

```text
F >= 65
K >= 2
S <= 3
股价接近支撑位
下跌缩量
分时出现承接
板块没有明显破位
```

买入比例：

| 信号强度 | T 仓买入比例 |
| --- | ---: |
| 弱信号 | 5% |
| 中等信号 | 10% |
| 强信号 | 15% - 20% |

正 T 买入后，不是必须当天卖出。

如果当天没有反弹，但买入逻辑没有破坏，可以次日处理。

但如果买入后跌破支撑，应执行止损，而不是变成长期加仓。

### 9.3 正 T 卖出规则

正 T 买入后，出现以下情况卖出等量底仓：

```text
反弹到压力位
上涨达到 0.8 - 1.5 个 ATR
分时冲高回落
放量但涨不动
红利板块转弱
大盘跳水
```

卖出目标：

```text
不是卖在最高点，而是完成成本下降。
```

### 9.4 倒 T 卖出规则

满足以下条件，可以考虑倒 T：

```text
股价接近压力位
短期涨幅较大
放量滞涨
K < 1.5
S 上升
G 没有继续增强
```

倒 T 卖出比例：

| 风险程度 | 卖出比例 |
| --- | ---: |
| 轻微压力 | 5% |
| 明显压力 | 10% |
| 高位滞涨 | 15% - 20% |

倒 T 后买回条件：

```text
回踩支撑
缩量止跌
分时承接恢复
板块没有退潮
```

如果卖出后继续上涨，不要强行追回。

因为倒 T 的失败处理是：

```text
承认少赚，不追高补错。
```

## 10. 评分系统设计

### 10.1 总评分

最终交易评分可以设计为：

```text
Total Score = 0.35F + 0.35R + 0.30T
```

其中：

```text
F = 基本面评分
R = 退神理论评分
T = 技术位置评分
```

### 10.2 退神理论评分 R

```text
R = 0.25G + 0.30Z + 0.25K + 0.20(5 - S)
```

解释：

```text
G 越高越好
Z 越高越好
K 越高越好
S 越低越好
```

每项 0 - 5 分。

| 分数 | 含义 |
| ---: | --- |
| 5 | 很强 |
| 4 | 偏强 |
| 3 | 中性 |
| 2 | 偏弱 |
| 1 | 很弱 |
| 0 | 无效 |

### 10.3 技术评分 T

| 项目 | 权重 |
| --- | ---: |
| 位置质量 | 35% |
| 成交量结构 | 25% |
| 趋势状态 | 20% |
| 分时承接 | 20% |

技术评分用于判断买卖点。

```text
T >= 80：技术位置好
T 60 - 79：可以观察
T < 60：不操作
```

## 11. 决策矩阵

### 11.1 买入 T 仓

| 条件 | 状态 |
| --- | --- |
| 基本面 F | >= 65 |
| 退神 R | >= 70 |
| 技术 T | >= 75 |
| 盈亏比 K | >= 2 |
| 卖压 S | <= 3 |
| 趋势 | 未破位 |

结论：

```text
允许买入 T 仓。
```

### 11.2 卖出 T 仓

| 条件 | 状态 |
| --- | --- |
| K | < 1.5 |
| S | 上升 |
| 技术位置 | 接近压力 |
| 成交量 | 放量滞涨 |
| 分时 | 冲高回落 |

结论：

```text
卖出 T 仓或做倒 T。
```

### 11.3 停止做 T

出现以下任意条件，停止做 T：

```text
F < 60
基本面逻辑证伪
行业景气明显下降
股价跌破中期趋势
反弹缩量，下跌放量
板块系统性走弱
连续两次 T 失败
```

### 11.4 减底仓

出现以下条件，减底仓：

```text
F < 55
分红逻辑恶化
业绩明显低于预期
行业周期反转向下
重大利空
长期均线破位后无法修复
红利资产整体退潮
```

## 12. 交易日执行流程

### 12.1 盘前检查

盘前检查 5 件事：

```text
1. 基本面有没有新变化？
2. 行业有没有新消息？
3. 大盘和红利板块环境如何？
4. 当前价格接近支撑还是压力？
5. 今天计划正 T、倒 T，还是不动？
```

输出结果：

```text
今日策略：
A. 只观察
B. 支撑位低吸
C. 压力位卖 T
D. 跌破止损
E. 不交易
```

### 12.2 盘中观察

盘中只看四个点：

```text
1. 是否到达计划价格？
2. 到达价格时量能是否正常？
3. 分时是否有承接？
4. 板块是否同步？
```

不因为一根分时拉升临时追涨。

### 12.3 盘后复盘

盘后记录：

```text
今日是否操作：
操作原因：
是否符合模型：
T 仓收益：
底仓变化：
成本变化：
失败原因：
明日计划：
```

重点不是今天赚没赚钱，而是：

```text
是否按模型执行。
```

## 14. 信号实现方式

### 14.1 人工版

适合初期。

每天只做一张表：

```text
F：基本面评分
G：关注度评分
Z：确定性评分
K：盈亏比评分
S：卖压评分
T：技术评分
```

然后根据规则执行。

优点：

```text
简单
灵活
适合培养盘感
```

缺点：

```text
主观性较强
容易情绪化
复盘依赖纪律
```

### 14.2 半自动版

用 Excel 或 Python 自动计算：

```text
均线
涨跌幅
成交量变化
支撑压力
ATR
盈亏比
评分结果
```

人工负责判断：

```text
基本面是否变化
行业逻辑是否变化
资金风格是否变化
```

适合你的当前阶段。

### 14.3 程序化版

程序每日生成信号：

```text
BUY_T
SELL_T
HOLD
REDUCE
STOP_T
```

但红利股做 T 不建议完全自动化。

原因：

```text
基本面和行业事件很难完全量化。
红利股受政策、分红、周期、市场风格影响较大。
```

更合理的是：

```text
程序给信号，人做最终确认。
```

## 15. 示例：中远海控模型化判断

假设你观察中远海控。

### 15.1 基本面层

你要先判断：

```text
当前运价是否改善？
公司利润是否稳定？
分红预期是否还在？
估值是否合理？
现金流是否安全？
```

如果这些问题大部分为正，则允许底仓。

如果运价转弱、利润恶化、分红预期下降，则不能因为“它是红利股”继续加仓。

### 15.2 退神理论层

你要判断：

```text
G：最近航运板块是否有资金回流？
Z：分红、运价、业绩预期是否增强？
K：当前位置距离支撑和压力是否划算？
S：前高附近是否有大量卖盘？
```

例如：

```text
股价回踩 20 日线
成交量缩小
航运板块没有破位
运价预期没有恶化
分红逻辑还在
```

这时：

```text
S 下降
K 改善
Z 稳定
```

可以考虑正 T。

反过来，如果：

```text
股价连续上涨
接近前高
放量但涨不动
航运板块开始分化
社交平台讨论变多
```

这时：

```text
K 下降
S 上升
G 可能快耗
```

适合卖 T 或倒 T。

## 16. 风控规则

### 16.1 单票仓位上限

红利股也不能无上限重仓。

建议：

```text
单只股票总仓位 <= 30%
单只周期股总仓位 <= 25%
T 仓单次操作 <= 总资金 10%
极端情况下预留现金 >= 10%
```

如果是 ETF，可以适当提高。

### 16.2 止损规则

做 T 的止损不是看浮亏多少，而是看逻辑是否失效。

止损条件：

```text
跌破支撑且无法收回
放量长阴
基本面逻辑恶化
板块破位
T 仓买入理由消失
```

不能把 T 仓亏损变成底仓。

这是纪律核心。

### 16.3 停手机制

出现以下情况，暂停交易：

```text
连续 2 次做 T 失败
大盘极端波动
标的发生重大利空
自己已经情绪化
当天没有计划却临时追涨杀跌
```

暂停后只复盘，不操作。

## 17. 模型的核心优势

这套模型的优势是：

```text
1. 不会因为短线波动丢掉底仓。
2. 不会因为长期逻辑盲目补仓。
3. 用基本面解决持有资格。
4. 用退神理论解决资金方向。
5. 用技术位置解决买卖时机。
6. 用仓位系统控制风险。
```

它比单纯技术分析更稳，因为有基本面锚。

它比单纯价值投资更灵活，因为可以利用波动降低成本。

它比政策主题短线更低风险，因为不依赖连续脉冲。

## 18. 模型的主要风险

### 18.1 趋势下跌中越 T 越亏

最大风险是：

```text
把下跌趋势误判为区间震荡。
```

解决方法：

```text
跌破中期趋势后停止做 T。
F 下降后停止做 T。
反弹缩量、下跌放量时停止做 T。
```

### 18.2 过度交易

做 T 容易变成手痒交易。

解决方法：

```text
没有计划不交易。
K < 2 不买 T。
不到压力位不卖 T。
单日最多操作 1-2 次。
```

### 18.3 用分红掩盖周期风险

高股息不等于安全。

周期股的高股息可能来自上一轮高利润。

如果行业进入下行周期，未来分红可能下降。

所以中远海控这类股票要重点看：

```text
未来利润
未来现金流
未来分红能力
```

而不是只看过去分红。

## 19. 最终执行口诀

这套模型可以压缩成：

```text
基本面没坏，才允许做 T；
支撑位低吸，压力位高抛；
缩量回踩可买，放量滞涨要卖；
上涨看买盘是否持续，下跌看卖压是否释放；
底仓赚长期逻辑，T 仓赚短期波动；
趋势破位不做 T，逻辑坏了减底仓。
```

更完整一点：

```text
底仓由基本面决定，
T 仓由盈亏比决定，
买点由支撑和卖压决定，
卖点由压力和获利盘决定，
是否继续操作由退神四变量决定。
```

## 20. 最终模型版本

### 长期 / 红利 / 做 T 模型 V1.0

```text
输入：
基本面数据、行业数据、行情数据、成交量、支撑压力、资金风格

处理：
基本面评分 F
退神理论评分 R
技术位置评分 T
仓位约束
风控规则

输出：
建底仓
持有
正 T 买入
正 T 卖出
倒 T 卖出
倒 T 买回
停止做 T
减底仓
清仓
```

最终决策逻辑：

```text
F 决定能不能持有；
R 决定资金行为是否支持；
T 决定买卖位置是否合适；
仓位系统决定买卖多少；
风控规则决定什么时候停止。
```

这就是模型核心：

```text
长期 / 红利 / 做 T
=
基本面逻辑定方向
+
退神理论看资金
+
技术位置做执行
```
